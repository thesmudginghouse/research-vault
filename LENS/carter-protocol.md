## WHAT THIS IS

The Carter Protocol is a **repeatable, domain-agnostic research methodology** for finding hidden signals in complex datasets. It was developed through UAP field investigation using the Subtraction Methodology and has been successfully applied across earth science, water intelligence, plasma physics, genomics, medical research, mathematics, and cybersecurity.

The core thesis: **the insight lives in the intersection, not the individual files.**

No single paper contains the finding. The finding emerges from cross-domain subtraction — when you hold the regulatory data AND the raw observations AND the methodology simultaneously, the gap becomes visible.

### STAGE 1 — SIGNAL DECLARATION
**Input:** John states the domain of interest and the question being asked

**AI outputs:**
- Restatement of the question in subtraction terms ("What are we trying to isolate?")
- Identification of what Category 1, 2, and 3 noise likely exists in this domain
- Targeted search queries for primary literature, journals, PhD theses, regulatory documents
- Data sources to seek (CSV, government databases, monitoring stations, sensor records)

**Quality standard:** Searches should target primary sources, not secondary summaries. The AI should seek the documentation closest to the raw observation.

---

### STAGE 2 — LITERATURE LOAD
**Input:** Papers, books, regulatory documents, manuals found via Stage 1 searches

**AI operations:**
- Identify the official consensus narrative in the domain
- Map what Category 2 factors (institutional, classified, suppressed) might exist
- Flag sources with identified industry incentives (Category 3 risk)
- Build a compressed domain knowledge base using the schema below

**Compression schema for each document:**
```
SOURCE: [title, author, year, DOI/URL]
DOMAIN_KEYWORDS: [5-10 terms]
CONSENSUS_CLAIM: [what this source asserts as established]
METHODOLOGY: [how they reached that claim]
DATA_USED: [what primary data underlies it]
GAPS_FLAGGED: [what this source explicitly does not explain]
CATEGORY_ASSESSMENT: [1/2/3 risk flags if any]
KEY_NUMBERS: [exact thresholds, measurements, values — preserved verbatim]
```

---

### STAGE 3 — DATA LOAD
**Input:** Raw data — CSV files, monitoring station records, observational logs, sensor outputs

**AI operations:**
- Do NOT interpret yet — describe what is in the data first
- Identify the measurement methodology and its known limitations
- Map the data against the literature loaded in Stage 2
- Flag any observations that fall outside the consensus range established in Stage 2
- Note any data that is ABSENT that should be present given the regulatory or scientific framework

**The absence of expected data is as significant as the presence of unexpected data.**

---

### STAGE 4 — SUBTRACTION
**Apply the methodology:**

1. Remove all observations explainable by Category 1 (known, prosaic, baseline)
2. Remove all observations explainable by Category 2 (institutional, suppressed, undisclosed)
3. Remove all observations that are Category 3 artifacts (industry-inflated, methodologically contaminated)
4. Document the residual

**The residual analysis:**
- What does the data say that the literature does not account for?
- Where does the regulatory framework assert X but the raw data shows Y?
- What patterns appear in the intersection of multiple data streams that appear in none individually?
- What is the smallest, most parsimonious explanation for the residual?

**Output format:**
```
RESIDUAL FINDING: [statement of what was found]
CONFIDENCE: [High/Medium/Low + rationale]
CATEGORY 4 ASSESSMENT: [does this qualify as genuine anomaly?]
NEXT INVESTIGATION STEP: [what data or analysis would confirm/deny]
PRACTICAL APPLICATION: [what decision or action does this finding support?]
```

## QUALITY STANDARDS

**What constitutes a valid finding:**
- Residual survives all four subtraction categories
- Finding is supported by at least two independent data streams
- Explanation is parsimonious (does not require exotic mechanisms if conventional ones explain it)
- Uncertainty is honestly documented — "we don't know" is a valid and valuable output

**What disqualifies a finding:**
- Based primarily on testimonial or secondary sources without primary data confirmation
- Certainty claimed beyond what the evidence supports
- Category 4 reached without exhausting Category 1, 2, and 3

**The honest position is always more valuable than the confident one.**
